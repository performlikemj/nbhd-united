Design System — Accessibility

Inclusive by design, not by afterthought

Every color passes WCAG AA. Every component is keyboard-navigable. Every interaction announces itself to assistive technology. This is v2.

Accessibility: All type sizes use rem (not px), respecting the user's browser font size preference. The minimum text size is 0.75rem (12px at default), ensuring readability. At 200% browser zoom, the layout reflows without horizontal scrolling.
04

Buttons

All buttons are keyboard-focusable with custom :focus-visible rings. Disabled buttons use aria-disabled for screen reader clarity.

♿ focus-visible
Variants — Tab through to see focus rings
Sizes
States
Keyboard: Tab to focus, Enter or Space to activate. Focus ring is a 2px var(--accent) outline with 2px surface-colored offset — visible on any background. Disabled buttons use both disabled attribute and aria-disabled="true".
05

Pattern: Interactive cards are <a> elements (not divs with click handlers). This gives them native keyboard focus, Enter activation, and correct screen reader semantics without needing tabindex or role="button".
06

Form Elements

Every input has an associated <label>. Hints use aria-describedby. Errors use aria-invalid and aria-errormessage.

♿ label + describedby
As it appears on your account
Please enter a valid email address
You can change this anytime
Maximum 500 characters
Preferences
Settings
Form patterns: Native <input type="checkbox"> is visually hidden but remains in the DOM for screen readers and keyboard access. The visual checkbox is aria-hidden="true". Toggles use role="switch" with aria-checked. Error states use aria-invalid="true" + aria-describedby pointing to the error message, which has role="alert" for live announcement.
07

Badges & Pills

Decorative dots are aria-hidden. Color is never the only indicator — text labels always accompany status color.

Active Error Warning Info Neutral Feature Premium Pending
Color independence: Status is always conveyed through text labels, never color alone. This ensures colorblind users and screen reader users receive the same information. Dots are decorative (aria-hidden="true").

Landmarks: Screen readers can jump between <nav>, <main>, and <footer> landmarks. Each nav uses a unique aria-label ("Design system sections" vs "Application navigation example") so users can distinguish them. aria-current="page" announces the active link.
09

Modal & Focus Trap

Focus is trapped inside the modal when open. Esc closes it. Focus returns to the trigger button on close.

♿ focus trap + aria-modal
Click or press Enter to open. Focus will be trapped inside.
Focus trap: When the modal opens, focus moves to the first focusable element inside. Tab cycles only within the modal. Shift+Tab wraps backwards. Esc closes and returns focus to the trigger. Background content gets aria-hidden="true" and inert.
